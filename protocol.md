# Urevo SpaceWalk 3S — BLE-протокол (разобрано с реального устройства)

Устройство: **URTM024**, производитель `UREVO`, прошивка `V90.08.12`.
Адрес (CoreBluetooth UUID на этом Маке): `B223F71E-3A43-9B2E-8331-4ECB1B52A426`.
Профиль: **стандартный FTMS** (Fitness Machine Service, `0x1826`) + сервис Device Information (`0x180a`) + два проприетарных сервиса (`0xfff0`, `0xfee0`, `5833ff01-…`) — для базовой работы не нужны.

## Карта GATT (главное)

| Сервис | Характеристика | Свойства | Назначение |
|--------|----------------|----------|------------|
| 0x1826 | **0x2acd** | notify | **Treadmill Data** — телеметрия (см. ниже) |
| 0x1826 | **0x2ad9** | write, indicate | **Fitness Machine Control Point** — управление |
| 0x1826 | 0x2acc | read | Fitness Machine Feature = `5c 16 00 00 03 00 00 00` |
| 0x1826 | 0x2ad4 | read | Supported Speed Range = min 1.00, max 6.00, шаг 0.10 км/ч |
| 0x1826 | 0x2ad5 | read | Supported Inclination Range = 0…9.0 %, шаг 1.0 % |
| 0x1826 | 0x2ad3 | read, notify | Training Status |
| 0x1826 | 0x2ada | notify | Fitness Machine Status |

## Разбор Treadmill Data (0x2acd), 26 байт

Флаги пакета = `0x259c`. Присутствуют поля: distance, inclination, elevation, energy, HR,
elapsed time, **+ проприетарное поле шагов** (флаг-бит 13, которого нет в стандарте FTMS —
Urevo дописал счётчик шагов в конец пакета).

Пример: `9c 25 5e 01 1e 0a 00 5a 00 00 00 00 00 00 00 31 01 00 00 00 00 7f 0e c7 10 00`

| Смещение | Байты | Поле | Значение в примере |
|----------|-------|------|--------------------|
| 0–1  | `9c 25` | flags | 0x259c |
| 2–3  | `5e 01` | **скорость**, uint16, 0.01 км/ч | 3.50 км/ч |
| 4–6  | `1e 0a 00` | **дистанция**, uint24, м | 2590 м |
| 7–8  | `5a 00` | наклон, sint16, 0.1 % | 9.0 % |
| 9–10 | `00 00` | угол рампы | 0 |
| 11–12 | `00 00` | подъём (+), 0.1 м | 0 |
| 13–14 | `00 00` | подъём (−) | 0 |
| 15–16 | `31 01` | **калории**, uint16, ккал | 305 |
| 17–18 | `00 00` | ккал/час | 0 |
| 19   | `00` | ккал/мин | 0 |
| 20   | `00` | пульс, uint8 | 0 (нет ремня) |
| 21–22 | `7f 0e` | **время сессии**, uint16, с | 3711 с |
| **23–24** | `c7 10` | **ШАГИ**, uint16 LE | **4295** ← то, что нужно |
| 25   | `00` | старший байт шагов (uint24) | 0 |

Проверка на реальных данных: за 59 с шаги 4295→4376 (+81), дистанция 2590→2650 м (+60),
скорость ~3.5 км/ч. Всё сходится: ~0.74 м/шаг, ~82 шага/мин. **Поле шагов подтверждено.**

## Управление (Control Point 0x2ad9) — стандартные команды FTMS

Записывать в 0x2ad9 (ответ приходит indicate). Порядок:

1. `0x00` — **Request Control** (обязательно первым).
2. `0x07` — Start / Resume.
3. `0x02 <lo> <hi>` — **Set Target Speed**, uint16 LE в 0.01 км/ч. Напр. 3.0 км/ч = 300 = `0x02 2c 01`.
4. `0x03 <lo> <hi>` — Set Target Inclination (0.1 %).
5. `0x08 0x01` — Stop. `0x08 0x02` — Pause.

**Slowdown** = периодически слать opcode `0x02` с убывающей целевой скоростью (шаг 0.1 км/ч,
диапазон 1.0–6.0). Управление скоростью с телефона — та же команда. Проверять на устройстве
осторожно (лента поедет) — только когда пользователь готов.

## Итог по фичам

- **Цель по шагам** — читаем шаги напрямую из 0x2acd[23:25]. ✅
- **Slowdown / контроль скорости** — стандартный FTMS Control Point 0x2ad9. ✅
- Реверс не понадобился: это чистый FTMS + одно вендорное поле шагов.
